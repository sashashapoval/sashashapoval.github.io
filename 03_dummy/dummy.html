Dummy variables in regressions

Motivation

To understand why dummy variables are essential in regression models, consider two simple artificial data-generating processes. In both cases, the outcome variable $y$ follows a linear pattern, but an underlying categorical factor changes the behavior of the model. Without explicitly including that factor in the regression, the model cannot capture the true structure of the data.

Example 1
  1. Model:
$$ y = \beta_0 + \beta_1 x + \varepsilon $$

but the intercept $\beta_0$ takes two distinct values. In the real-world, this behavior can caused by differences between men and women, distinct geographical locations, and other binary characteristics.

  1. The true relationship between $y$ and $x$ consists of two parallel lines with the same slope $\beta_1$, but with different vertical shifts.

  2. If we fit a simple linear regression without acknowledging the two intercepts, the model is forced to estimate a single intercept. As a result, it averages the two groups and misrepresents both.

  3. Let's construct a numerical example, altering the intercept between two values with the probability of $1/2$.

import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
import statsmodels.formula.api as smf

# Parameters
b = 2
intercept1 = -5
intercept2 = 5
ndata = 200

# Generate data
x = np.random.uniform(0, 10, ndata)
group = np.random.randint(0, 2, ndata)
intercepts = np.where(group == 1, intercept1, intercept2)
y = b * x + intercepts + np.random.normal(size=ndata)

# Build dataframe
df_artificial = pd.DataFrame({
    "x": x,
    "y": y,
    "group": group
})

# Fit regression on THIS dataset
reg = smf.ols("y ~ x", data=df_artificial).fit()

# Plot

plt.scatter(df_artificial["x"], df_artificial["y"], s=10)

# Best-fit line
x_vals = np.linspace(df_artificial["x"].min(), df_artificial["x"].max(), 200)

plt.plot(x_vals, y_pred, color="red", linewidth=2, label="Best fit")
plt.xlabel("x")
plt.ylabel("y")
plt.title("Generated Data with Regression Line")

plt.show()

Solution:

$$ y = \beta_0 + \beta_1 x + {\color{red}{\gamma D}} + u $$

where $D$ is a dummy that equals 1 for group A and 0 for group B.

    #Function formatting the regression outcome; already appeared in the multiple regression notebook
def format_regression_output(results):
    """
    Print a statsmodels OLS regression in Stata-style:
    - Model equation with coefficients
    - Coefficients with standard errors underneath
    - t-statistics and p-values
    - Number of observations, SSR, and R-squared
    """

    params = results.params
    ses = results.bse
    t_stats = results.tvalues
    p_values = results.pvalues
    dep_var = results.model.endog_names

    # ----- Build regression equation -----
    eq = f"{dep_var} = "

    # Intercept first (if present)
    if "Intercept" in params.index:
        eq += f"{params['Intercept']:.4f}"
    else:
        eq += "0"

    # Add other variables
    for var in params.index:
        if var == "Intercept":
            continue
        sign = "+" if params[var] >= 0 else "-"
        eq += f" {sign} {abs(params[var]):.4f}*{var}"

    # ----- Print equation -----
    print("Model equation:\n")
    print("  " + eq + "\n")

    # ----- Print coefficients + SEs + t-statistics + p-values -----
    print("Coefficients (standard errors in parentheses, t-statistics and p-values in brackets):\n")
    for var in params.index:
        print(f"{var:12} {params[var]: .4f}")
        print(f"{'':12} ({ses[var]:.4f}) [t={t_stats[var]:.3f}, p={p_values[var]:.3f}]")
    print()

    # ----- Print model statistics -----
    n = int(results.nobs)
    ssr = float((results.resid ** 2).sum())
    r2 = float(results.rsquared)

    print(f"Number of observations (n): {n}")
    print(f"Sum of squared residuals (SSR): {ssr:.4f}")
    print(f"R-squared: {r2:.4f}\n")

    Explanation: What does pt.dmatrices do?

    patsy.dmatrices() takes a model formula (written in R-style syntax) and a DataFrame, and then:

    1. Parses the formula into a response variable (y) and a design matrix of regressors (X).
    2. Applies all formula operations (e.g., +, *, interaction terms, polynomial terms, C() for categories, etc.).
    3. Returns the resulting y and X as matrices or DataFrames (here, because return_type='dataframe', you get pandas DataFrames).

    So the line:

    y, X = pt.dmatrices('np.log(wage) ~ educ + exper + tenure', data=wage1, return_type='dataframe')

    create the dependent variable log(wage) and the matrix of independent variables (intercept, educ, exper, tenure).

    What does smo.variance_inflation_factor do?

    variance_inflation_factor() computes the Variance Inflation Factor (VIF) for a specific regressor in the design matrix.

    For each column i in the matrix X, VIF is defined as:

    $$ \text{VIF}_i = \frac{1}{1 - R_i^2}, $$

    where $ R_i^2 $ is the R-squared from a regression of $ X_i $ on all the other columns of X.

    Interpretation:

    • VIF = 1 no multicollinearity
    • VIF between 1 and 5 mild to moderate multicollinearity
    • VIF > 10 high multicollinearity (often considered problematic)

    Thus, in the loop:

    for i in range(K):
    VIF[i] = smo.variance_inflation_factor(X.values, i)

    each regressor (including the intercept) is evaluated for how strongly it is linearly related to the others.

    Linear Probability Model

    In the linear probability model (LPM), the dependent variable $y$ is binary, and the regression model is

    $$ y_i = \beta_0 + \beta_1 x_{i1} + \beta_2 x_{i2} + \cdots + \beta_k x_{ik} + u_i. $$$$ \Exp(y \mid x) = \beta_0 + \beta_1 x_{i1} + \beta_2 x_{i2} + \cdots + \beta_k x_{ik}. $$$$ \Prob(y=1 \mid x) = \beta_0 + \beta_1 x_{i1} + \beta_2 x_{i2} + \cdots + \beta_k x_{ik}. $$

    Testing the Hypothesis $H_0: \beta_1 = 0$ against the two-sided alternative

    To test the hypothesis

    $$ H_0: \beta_1 = 0 \quad \text{vs} \quad H_1: \beta_1 \ne 0, $$

    you must use heteroskedasticity-robust standard errors (HC0, HC1, HC2, or HC3).
    With robust standard errors, OLS is still consistent and inference is valid.

    The robust $t$-statistic is:

    $$ t = \frac{\hat{\beta}_1}{\text{SE}_{\text{robust}}(\hat{\beta}_1)}. $$

    The two-sided p-value is:

    $$ p = 2 (1 - F_{t}(|t|)), $$

    where $F_t$ is the appropriate reference distribution (asymptotically standard normal when sample size is large).

    Notes

    • You cannot use regular OLS standard errors.
    • You can test hypotheses using robust standard errors.
    • Alternatively, you may estimate a nonlinear model (logit/probit), but this is not required to test $\beta_1 = 0$ in the LPM.

    Logit model

    The logit model is a nonlinear regression model used for a binary dependent variable. It specifies that the probability that $Y = 1$ given regressors $X$ follows a logistic function:

    $$ \Prob{(Y = 1 \mid X)} = \Lambda(X\beta), $$

    where the logistic CDF is

    $$ \Lambda(z) = \frac{e^{z}}{1 + e^{z}}, $$

    Equivalently, the logit model can be written in terms of log-odds:

    $$ \log\left( \frac{\Pr(Y = 1 \mid X)}{\Pr(Y = 0 \mid X)} \right) = X\beta. $$

    The parameters $\beta$ are typically estimated by maximum likelihood.

    Maximum likelihood estimate of $\beta$

    The log-likelihood for a sample of size n is:

    $$ \ell(\beta) = \sum_{i=1}^n \left[\, Y_i \ln p_i \;+\; (1 - Y_i)\ln(1 - p_i) \right]. $$

    Substituting the logistic function $p_i = e^{X_i } / (1 + e^{X_i })$, we get:

    $$ \ell(\beta) = \sum_{i=1}^n \left[ Y_i X_i \beta - \ln(1 + e^{X_i \beta}) \right]. $$

    The maximum likelihood estimator (MLE) is defined as:

    $$ \hat{\beta} = \arg\max_\beta \ \ell(\beta). $$

    There is no closed-form expression for $\hat{\beta}$.
    It is obtained by solving the first-order condition:

    $$ \frac{\partial \ell(\beta)}{\partial \beta} = \sum_{i=1}^n (Y_i - p_i) X_i = 0. $$

    The solution is computed using iterative numerical methods such as NewtonRaphson or Fisher scoring.

    \hat{\beta} in Python

    In Python, the maximum likelihood estimate (MLE) \hat{\beta} for the logit model is typically computed using the statsmodels package. The optimizer solves the likelihood equations numerically (usually via NewtonRaphson or a related algorithm).

    import statsmodels.api as sm

X = sm.add_constant(X)       # X : matrix of regressors, y :he binary dependent variable; adds the intercept

    Testing the hypothesis: $H_0:\ \beta_1 = 0$ against $H_1:\ \beta_1 \ne 0$

    In a logit model estimated by maximum likelihood, hypothesis tests for coefficients are typically done using the Wald test, which is based on the estimated coefficient and its standard error.

    Let $\hat{\beta}_1$ be the MLE of $\beta_1$, and let $se(\hat{\beta}_1)$ be its standard error.

    The Wald test statistic is:

    $$ z = \frac{\hat{\beta}_1}{se(\hat{\beta}_1)}. $$
    Standard Error

    is the square root of the diagonal element of the covariance matrix of the MLE:

    $$ \text{se}(\hat{\beta}_j) = \sqrt{\Var(\hat{\beta}_j)}. $$

    The covariance matrix is the inverse of the observed Fisher information matrix:

    $$ \text{Cov}(\hat{\beta}) = I(\hat{\beta})^{-1}, $$

    where the Fisher information matrix $I(\beta)$ for the logit model is

    $$ I(\beta) = \sum_{i=1}^{n} p_i (1 - p_i) \, X_i X_i^\top, $$

    with $$ p_i = \frac{e^{X_i^\top \beta}}{1 + e^{X_i^\top \beta}}. $$

    Under the null hypothesis (for large samples), the Wald statistic is approximately standard normal:

    $$ z \sim N(0,1). $$

    To test $$ H_0: \beta_1 = 0 \quad \text{vs} \quad H_1: \beta_1 \ne 0, $$ against the two-sided hypothesis you compute the p-value: $$ p = 2 \big(1 - \Phi(|z|)\big), $$

    and reject $H_0$ if $p$ is below your chosen significance level (e.g., 0.05).

    $$ H_0: \beta_1 = 0. $$

    If the p-value is small, you reject the null in favor of the two-sided alternative.

    In a logit model, the Wald test for a coefficient $\beta_j$ is based on the estimated coefficient $\hat{\beta}_j$ and its standard error $\text{SE}(\hat{\beta}_j)$:

    $$ z = \frac{\hat{\beta}_j}{\text{se}(\hat{\beta}_j)}. $$

    Marginal Effects and Key Relationships

    In a logit model, we often distinguish between the logit coefficients and the marginal effects.

    1. Marginal Effects (mfx)

    The marginal effect of a variable like educ measures the change in the probability of labor-force participation associated with a one-unit increase in education:

    $$ \text{ME}_{\mathtt{educ}} = \frac{\partial P(\mathtt{inlf}=1)}{\partial\, \mathtt{educ}} $$
    • Computed by results_logit.get_margeff(at='overall') in Python.
    • Interpretation: If $\text{ME}_{\mathtt{educ}} = 0.05$, a one-year increase in education increases the probability of participating in the labor force by 5 percentage points, holding other variables constant.
